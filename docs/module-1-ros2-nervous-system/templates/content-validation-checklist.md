# Content Validation Checklist

Before publishing any ROS 2 content, ensure it meets these criteria:

## Technical Accuracy
- [ ] All code examples follow official ROS 2 documentation standards
- [ ] All concepts align with ROS 2 Humble Hawksbill (or specified version)
- [ ] All examples are tested and functional
- [ ] All URDF examples are valid and properly structured

## Educational Quality
- [ ] Content is appropriate for AI students and developers with Python background
- [ ] Concepts are explained clearly before showing code examples
- [ ] Each section has practical, runnable examples
- [ ] Troubleshooting sections are included for common issues

## Documentation Standards
- [ ] All pages have proper frontmatter (title, sidebar_position)
- [ ] All code blocks have appropriate language identifiers
- [ ] All external links are verified
- [ ] All cross-references are valid
